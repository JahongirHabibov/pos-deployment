# POS-система — Руководство по развёртыванию в продакшн

Данное руководство описывает процесс установки системы точки продаж на сервере клиента с использованием Docker Compose.

---

## Предварительные требования

- Linux-сервер с установленными **Docker** и **Docker Compose**
- Учётные данные, предоставленные разработчиком/дистрибьютором или администратором Legisell:
  - Имя пользователя GHCR и токен с правами только на чтение
  - URL бэкенда Legisell и одноразовый токен для провизии

---

## Шаг 1 — Авторизация в приватном реестре контейнеров

Docker-образы размещены на GitHub Container Registry (GHCR). Перед загрузкой образов необходимо выполнить вход:

```bash
export GHCR_USER="<ваш-ghcr-пользователь>" && \
export GHCR_TOKEN="<ваш-ghcr-readonly-токен>" && \
echo "$GHCR_TOKEN" | sudo docker login ghcr.io -u "$GHCR_USER" --password-stdin
```

> Все учётные данные предоставляются разработчиком/дистрибьютором или администратором Legisell.

---

## Шаг 2 — Провизия файла `.env`

Секреты (пароли базы данных, ключи API и т. д.) централизованно управляются в **Legisell License Manager**. Для каждого тенанта там генерируется одноразовый токен провизии, который скрипт `provision.py` использует для автоматического заполнения файла `.env`.

### 2.1 — Запуск скрипта провизии

```bash
python3 provision.py \
  --token <ONE_TIME_PROVISIONING_TOKEN> \
  --api-url <LEGISELL_BACKEND_URL>
```

**Необязательные аргументы:**

| Аргумент | По умолчанию | Описание |
|---|---|---|
| `--env-example` | `.env.example` | Путь к шаблону `.env` |
| `--env-output` | `.env` | Путь для сгенерированного файла `.env` |

Скрипт выполняет следующие действия:
1. Обращается к API Legisell, потребляет токен и получает секреты.
2. Копирует `.env.example` → `.env` (с резервной копией существующего `.env`).
3. Заменяет совпадающие ключи в `.env` полученными значениями и добавляет дополнительные секреты в конец файла.

### 2.2 — Установка тегов Docker-образов

После генерации файла `.env` необходимо указать корректные теги образов в переменных `IMAGE_*`. Эти значения предоставляются разработчиком/дистрибьютором:

```dotenv
IMAGE_BACKEND=ghcr.io/<org>/pos-backend:<tag>
IMAGE_FRONTEND=ghcr.io/<org>/pos-frontend:<tag>
IMAGE_IMAGE_SERVICE=ghcr.io/<org>/pos-image-service:<tag>
```

Откройте `.env` в любом текстовом редакторе и обновите эти значения.

---

## Шаг 3 — Запуск стека

После того как файл `.env` полностью заполнен, запустите все сервисы:

```bash
sudo docker compose -f docker-compose.prod.yml up -d
```

Docker Compose запустит следующие сервисы:

| Сервис | Описание |
|---|---|
| `pos-database` | PostgreSQL 17 — хранение данных |
| `pos-redis` | Redis 8 — слой кэширования |
| `pos-backend` | Сервер приложений FastAPI |
| `pos-frontend` | React PWA под управлением Nginx (публичная точка входа) |
| `pos-image-service` | Сервис загрузки изображений и генерации миниатюр |

Фронтенд доступен на порту, заданном переменной `POS_PUBLIC_PORT` в файле `.env`.

---

## Полезные команды

```bash
# Просмотр запущенных контейнеров и их состояния
sudo docker compose -f docker-compose.prod.yml ps

# Потоковый вывод логов всех сервисов
sudo docker compose -f docker-compose.prod.yml logs -f

# Потоковый вывод логов конкретного сервиса (например, backend)
sudo docker compose -f docker-compose.prod.yml logs -f backend

# Остановка стека (данные сохраняются в Docker-томах)
sudo docker compose -f docker-compose.prod.yml down

# Остановка с удалением всех томов данных (деструктивно — используйте осторожно)
sudo docker compose -f docker-compose.prod.yml down -v
```

## Устранение неполадок

| Симптом | Вероятная причина | Решение |
|---|---|---|
| `pull access denied` при запуске docker compose | Не выполнен вход в GHCR или токен истёк | Повторите Шаг 1 |
| `API request failed (HTTP 401/403)` из provision.py | Недействительный или уже использованный токен провизии | Запросите новый токен у администратора Legisell |
| Контейнер сразу завершает работу | Отсутствующие или некорректные значения в `.env` | Проверьте `docker compose logs <service>` и содержимое `.env` |
| Фронтенд недоступен | Порт `POS_PUBLIC_PORT` заблокирован брандмауэром | Откройте нужный порт в настройках брандмауэра/группы безопасности сервера |
